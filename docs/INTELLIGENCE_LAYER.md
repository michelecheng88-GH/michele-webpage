# Intelligence Layer

## Messy Inputs
- Visitor's 10 quiz answers (scale 1–4 per question)
- Free-text "biggest challenge" description
- Company size / sector (inferred from role + company name)

## Auto-Structure Schema (quiz answer payload)
```json
{
  "q1": 2, "q2": 3, "q3": 1, "q4": 4, "q5": 2,
  "q6": 3, "q7": 1, "q8": 4, "q9": 2, "q10": 3,
  "dimension_scores": {
    "S": 14, "A": 10, "F": 18, "E": 8, "R": 12
  },
  "total": 62,
  "band": "Emerging Adopter"
}
```

## Events to Track
- `quiz.started` — visitor hits /quiz
- `quiz.completed` — all 10 answers submitted
- `lead.captured` — form submitted with email
- `blog.viewed` — post slug + referrer
- `cta.clicked` — which button (quiz / call / service)

## Scoring Rules (v1 — rule-based, no AI)
- Each question scored 1–4; 2 questions per S.A.F.E.R. dimension
- Dimension max = 8; total max = 40 → normalised to 0–100
- Bands: 0–39 = Foundation Builder; 40–59 = Emerging Adopter; 60–79 = Strategic Integrator; 80–100 = AI-Ready Leader
- Recommendations: 3 static strings per band, selected by band key (lookup table in code)

## What Gets Ranked
- v1: leads sorted by `created_at` desc in admin panel
- Next: leads ranked by quiz score (higher score = closer to buying)
- Later: AI assigns a "fit score" based on challenge text + sector + score band

## v1 vs Later
- **v1:** all scoring is pure TypeScript, zero AI calls
- **Later:** `ai_narrative` generated by GPT-4o after lead capture; stored with source + confidence + review_status; Michele reviews before it influences any outbound action
